# abyo-speculate Benchmarks

Honest, reproducible numbers. We do not quote a single "X× faster" headline —
SD speedup depends heavily on the target/draft pair, the task, and the chosen
`draft_lookahead`.

## Hardware

| Component | Spec |
|-----------|------|
| GPU | NVIDIA GeForce RTX 4070 Ti SUPER (16380 MiB, compute 8.9 / Ada Lovelace) |
| Driver | 595.58.03 |
| CUDA toolkit | 12.4.131 |
| candle | 0.8.4 |
| dtype | BF16 (`--features cuda`) |

## Reproducing

Each row below was generated by:

```sh
cargo run --release --features cuda --bin abyo-speculate-bench -- \
    --target <TARGET> --draft <DRAFT> \
    --method both --max-tokens 128 --warmup 1 --runs 3 --draft-lookahead <K>
```

Models are pulled from Hugging Face on first run and cached at
`~/.cache/huggingface/hub`.

## Headline: target × draft pairs (k = 4)

128 new tokens, temperature 0.7, top_p 0.95, prompt = the default
"explain SD to a senior engineer" prompt (44 tokens).

| Target | Draft | AR tok/s | SD tok/s | Speedup | Notes |
|--------|-------|----------|----------|---------|-------|
| Qwen 2.5 1.5B | Qwen 2.5 0.5B | 60.4 | 59.7 | 0.99× | Param ratio (3×) too small to win |
| Qwen 2.5 3B | Qwen 2.5 0.5B | **34.3** | **49.0** | **1.43×** | Sweet spot for this GPU |
| Qwen 2.5 7B | Qwen 2.5 0.5B | 18.5 | OOM | N/A | Both models don't fit in 16 GB |

The 1.5B/0.5B pair shows the lower-bound: when target ≈ draft, SD's per-round
overhead eats the parallelisation win. The 7B/0.5B pair would likely show the
biggest speedup but doesn't fit alongside the draft on a 16 GB card without
quantisation.

## `draft_lookahead` sweep on Qwen 2.5 3B + 0.5B draft

Same 128-token / 3-run setup, varying `k` only:

| k | AR tok/s | SD tok/s | Speedup |
|---|----------|----------|---------|
| 2 | 35.2 | 47.8 | 1.36× |
| 3 | 35.1 | 47.7 | 1.36× |
| **4** | **34.3** | **49.0** | **1.43×** |
| 6 | 34.4 | 31.8 | 0.92× |
| 8 | 30.5 | 28.8 | 0.95× |

`k = 4` is optimal for this pair — beyond that, the per-round acceptance rate
drops fast enough that wasted draft work outweighs the parallelisation. This
matches the qualitative analysis in Leviathan et al. 2023 §3.4.

## What's NOT in this table (yet)

- **Per-task breakdowns** (chat / coding / translation / long context).
  The bench CLI takes `--prompt`; we'll publish a multi-task table once
  representative prompts are nailed down.
- **Real Medusa numbers.** The Medusa pipeline runs end-to-end on real
  Qwen with synthetic-init heads (correctness only — random heads always
  reject). Public Medusa checkpoints target Vicuna / Llama 2, not Qwen.
  Either pointing the loader at one of those or training a Qwen-Medusa
  head is the next deliverable.
- **EAGLE-2 / EAGLE-3.** Not implemented yet.
- **F16 / Q4 quantisation paths.** Would unlock 7B target on a 16 GB card.
- **Larger GPUs.** 24 GB+ cards (e.g. RTX 4090, A10G, L4) should land
  the 7B speedup easily.

## Honesty notes

- These numbers are **single-user, batch size 1**. Multi-batch SD is a
  different regime where vLLM / SGLang win and abyo-speculate is not the
  right tool.
- `tok/s` excludes prompt processing. The bench warmup absorbs first-token
  latency.
- BF16 on Ada is more or less a no-op accuracy hit vs F32. On Pascal /
  Volta it would degrade more.
- We do **not** assert that SD output is bit-identical to AR output —
  rejection sampling produces samples from the same target distribution,
  but trajectory-level you'd need temperature 0 + matching seeds.
