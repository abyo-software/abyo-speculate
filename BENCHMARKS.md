# abyo-speculate Benchmarks

Honest, reproducible numbers. We do not quote a single "X× faster" headline —
SD speedup depends heavily on the target/draft pair, the task, and the chosen
`draft_lookahead`.

| Component | Spec |
|-----------|------|
| GPU | NVIDIA GeForce RTX 4070 Ti SUPER (16380 MiB, compute 8.9 / Ada Lovelace) |
| Driver | 595.58.03 |

## Reproducing

Each row was generated by:

```sh
cargo run --release --features cuda --bin abyo-speculate-bench -- \
    --target <TARGET> --draft <DRAFT> --family <FAMILY> \
    --task <TASK> --method both \
    --max-tokens 128 --warmup 1 --runs <N> --draft-lookahead <K>
```

Models are pulled from Hugging Face on first run and cached at
`~/.cache/huggingface/hub`. `--family auto` (default) infers from the repo id.

## Per-task: Qwen 2.5 3B + 0.5B draft, k=4

128 new tokens, temperature 0.7, top_p 0.95, 1 warmup + 2 runs each.

| Task | AR tok/s | SD tok/s | Speedup |
|------|---------:|---------:|--------:|
| chat | 34.0 | 48.5 | **1.42×** |
| **code** | **33.9** | **59.8** | **1.76×** |
| translation | 33.8 | 47.2 | **1.40×** |
| long_context | 31.2 | 48.2 | **1.55×** |

`code` wins by a wide margin — coding tokens are highly predictable
(whitespace, brackets, common keywords), so the draft model's acceptance
rate is highest. `translation` and `chat` cluster around 1.4×; `long_context`
lands in between, with autoregressive baseline slightly slower (KV cache
reads dominate).

## Per-pair: target × draft (default chat task, k=4)

128 new tokens, 3 measured runs after 1 warmup.

| Target | Draft | AR tok/s | SD tok/s | Speedup | Notes |
|--------|-------|---------:|---------:|--------:|-------|
| Qwen 2.5 1.5B | Qwen 2.5 0.5B | 60.4 | 59.7 | 0.99× | Param ratio (3×) too small to win |
| **Qwen 2.5 3B** | **Qwen 2.5 0.5B** | **34.3** | **49.0** | **1.43×** | Sweet spot for this GPU |
| Qwen 2.5 7B | Qwen 2.5 0.5B | 18.5 | OOM | N/A | Both models don't fit in 16 GB |

The 1.5B/0.5B pair shows the lower-bound: when target ≈ draft, SD's
per-round overhead eats the parallelisation win. The 7B/0.5B pair would
likely show the biggest speedup but doesn't fit alongside the draft on a
16 GB card without quantisation.

## `draft_lookahead` sweep on Qwen 2.5 3B + 0.5B

Same 128-token / 3-run setup, varying `k` only:

| k | AR tok/s | SD tok/s | Speedup |
|---|---------:|---------:|--------:|
| 2 | 35.2 | 47.8 | 1.36× |
| 3 | 35.1 | 47.7 | 1.36× |
| **4** | **34.3** | **49.0** | **1.43×** |
| 6 | 34.4 | 31.8 | 0.92× |
| 8 | 30.5 | 28.8 | 0.95× |

`k = 4` is optimal for this pair — beyond that, the per-round acceptance
rate drops fast enough that wasted draft work outweighs the
parallelisation. This matches the qualitative analysis in Leviathan et al.
2023 §3.4.

## Medusa: loader compatibility

`tests/with_real_medusa_heads.rs` validates that
`MedusaHeadsCandle::from_fasterdecoding_pt` correctly parses
`FasterDecoding/medusa-vicuna-7b-v1.3`'s `medusa_lm_head.pt` (5 heads,
4096 hidden, 32000 vocab, BF16). Forward through the loaded heads
produces non-NaN logits of the expected shape.

A full Medusa speedup measurement would need either:

- The Vicuna 7B base model in safetensors format (the upstream `lmsys/`
  repo only ships PyTorch sharded `.bin`, and the `TheBloke/...-safetensors`
  mirrors are gated behind an HF token), **or**
- Q4 quantisation so 7B + heads fit in <16 GB.

Both are tracked under "future work" — for now, the loader is verified
against the real published format.

## What's NOT in this table (yet)

- **Real Medusa speedup numbers** — see above.
- **EAGLE-2 / EAGLE-3.** Not implemented yet.
- **F16 / Q4 quantisation paths.** Would unlock 7B target on a 16 GB card.
- **Larger GPUs.** 24 GB+ cards (e.g. RTX 4090, A10G, L4) should land
  the 7B speedup easily.

## Honesty notes

- These numbers are **single-user, batch size 1**. Multi-batch SD is a
  different regime where vLLM / SGLang win and abyo-speculate is not the
  right tool.
- `tok/s` excludes prompt processing. The bench warmup absorbs first-token
  latency.
- BF16 on Ada is a near-no-op accuracy hit vs F32. On Pascal / Volta it
  would degrade more.
- We do **not** assert that SD output is bit-identical to AR output —
  rejection sampling produces samples from the same target distribution,
  but trajectory-level you'd need temperature 0 + matching seeds.
